DriveToWeb links

DriveToWeb enables you to host websites on your favorite cloud drive. To access a file on your cloud drive, make it viewable by everyone, then visit https://drv.tw/~[username]/[drive]/[path] where

[username] is the account email address to access cloud drive,
[drive] is "gd" for Google Drive and "od" for Microsoft OneDrive, and
[path] is the full, case-sensitive path that locates the file.

An example using Google Drive

For example, to view the file index.html in top-level folder Web under account me@gmail.com on Google Drive ("gd"), shown as the following,

visit https://drv.tw/~me@gmail.com/gd/Web/index.html. Note that a wiggly "~" (tilde, not dash) is needed before [username]. You can access any public file on your Google Drive in a similar way.

If you omit the file name and use only https://drv.tw/~me@gmail.com/gd/Web/, the system will look for the file named "index.html" implicitly.

A DriveToWeb URL is case sensitive, i.e., upper-/lower-case letters are treated differently. In the example above, https://drv.tw/~me@gmail.com/gd/web/index.html (with lower-case "web") or https://drv.tw/~me@gmail.com/gd/WEB/index.html (with all-caps "WEB") will not work. It must exactly match the folder name, which is "Web".

Only public files can be viewed. We recommend keeping all files for the website in one folder and share it for public viewing. Subsequent files added to the folder are automatically shared.

You can make a file or folder public via Share > Advanced > Who has access (Change...).
